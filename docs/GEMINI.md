# Contexto do Gemini: `docs`

Este diretório é o repositório central para toda a documentação do projeto `gestao_updev`.

## Propósito

Manter uma documentação clara, concisa e atualizada é crucial para a colaboração, manutenção e onboarding de novos desenvolvedores.

## Conteúdo

*   **`api.yaml`:** A especificação OpenAPI (Swagger) que define o contrato da API REST. Este é o arquivo fonte da verdade para os endpoints.
*   **Documentos de Arquitetura:** Arquivos como `arquitetura-backend.md` e `FRONTEND_ARCHITECTURE.md` que descrevem as decisões de alto nível sobre a estrutura da aplicação.
*   **Padrões e Convenções:** Arquivos como `CODE_STANDARDS.md` e `padroes-codigo.md` que definem as diretrizes de codificação para manter a consistência.
*   **Guias de Desenvolvimento:** Documentos como `DEVELOPMENT.md` e `backend-setup.md` que ajudam os desenvolvedores a configurar e executar o projeto localmente.
*   **Planos Estratégicos:** Documentos como `TESTING_AND_SECURITY_PLAN.md` que delineiam os próximos passos e melhorias para o projeto.

Ao fazer alterações significativas na arquitetura, nos padrões ou nos processos, o desenvolvedor deve atualizar ou criar a documentação correspondente neste diretório.
