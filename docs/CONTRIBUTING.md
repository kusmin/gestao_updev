# Guia de Contribuição

Este documento descreve como contribuir para o projeto, desde o reporte de bugs até o envio de pull requests.

## Reportando Bugs

- **Verifique se o bug já não foi reportado:** Pesquise na [issue tracker](https://github.com/user/repo/issues).
- **Se não houver, crie uma nova issue:** Forneça um título claro, uma descrição detalhada do problema, passos para reproduzi-lo e, se possível, o ambiente (versão do sistema, navegador, etc.).

## Sugerindo Melhorias

- **Use a issue tracker para sugestões:** Crie uma issue com a tag `enhancement`.
- **Descreva a melhoria:** Explique o que você gostaria de ver e por que isso seria útil para o projeto.

## Processo de Desenvolvimento

1. **Faça um fork do repositório.**
2. **Crie uma nova branch para sua feature:** `git checkout -b feature/nome-da-feature`.
3. **Implemente suas alterações:** Siga os padrões de código do projeto.
4. **Adicione testes:** Garanta que suas alterações estão cobertas por testes.
5. **Envie um pull request:** Descreva suas alterações e aguarde a revisão.

## Padrões de Commit

- **Use o padrão Conventional Commits:** Ex: `feat: adiciona agendamento de serviços`.
- **Tipos permitidos:** `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`.
